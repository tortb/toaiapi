# Security Policy

## 报告安全漏洞

如果你发现 ToAIAPI 存在安全漏洞，请通过以下方式报告：

**请不要在公开的 GitHub Issue 中报告安全漏洞。**

### 报告方式

发送邮件至：security@toaiapi.com

### 报告内容

请包含以下信息：

1. 漏洞描述
2. 复现步骤
3. 影响范围
4. 建议修复方案（如有）

### 响应时间

- **P0（数据泄露、支付漏洞）**: 立即响应
- **P1（认证绕过、权限提升）**: 1 小时内响应
- **P2（XSS、CSRF）**: 24 小时内响应
- **P3（信息泄露、配置错误）**: 72 小时内响应

## 安全措施

### 数据安全

- 密码使用 Argon2id 加密存储
- API Key 使用 Argon2id 加密存储
- 敏感数据传输使用 HTTPS
- 敏感信息不记录到日志

### 支付安全

- 支付回调验证签名
- 使用 timingSafeEqual 比较签名
- 订单号唯一约束
- 支付状态只能通过回调修改

### API 安全

- 所有输入使用 class-validator 校验
- SQL 查询使用 Prisma 参数化
- 配置 CORS 和安全头
- 实施限流保护

### 基础设施安全

- 强制 HTTPS
- 配置 HSTS
- 使用安全的环境变量管理
- 定期更新依赖
